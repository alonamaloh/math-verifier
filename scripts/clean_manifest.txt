# clean_manifest.txt — the highest-standard set: files that READ LIKE MATH.
#
# Entry rule: a file joins this list only after someone has READ the proof and
# judged it reads like a textbook — NOT merely because it passes the ratchets.
# The two ratchets (`make clean-check` = section-1 CIC leaks; `make
# clean-anon-ratchet` = anonymous connective tuples) are the automated FLOOR and
# the regression guard, but they are necessary-not-sufficient: they do not see
# pattern-match `cases <value>` or `successor` outside Natural/, so a file can
# score 0/0 and still read like machine output. Adding a line here asserts the
# read happened. See docs/CLEAN_STYLE_PLAN.md and the memory
# `read_proofs_linter_is_a_tool` / `clean_gate_necessary_not_sufficient`.
#
# `scripts/clean_status.py` prints a per-file successor-outside-Natural advisory
# (a high count = re-read this proof). This is the single source of truth; there
# is deliberately NO separate "reviewed" list.
#
# Grows toward whole-library coverage. Seeded from the original headliner cone
# (the former "baby" set); some seed/score-only entries still owe a confirming
# read (worklist = the successor advisory, high counts first).
